v-if在切换时会尽可能的复用相同的元素，可以在元素上添加不同的key来完全独立渲染，没有添加key的元素依然会尝试复用

v-show仅仅是控制元素的display属性，不显示时依然存在于dom中，v-show不支持template语法，不支持v-else

一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

不推荐v-if和v-for同时使用，v-for具有更高的优先级
